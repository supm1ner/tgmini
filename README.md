# Telegram Mini-Bot — Соревнование по минимизации

## Вариант 1: Ультра (рекомендую) — 30 байт
**Файл:** `loader3.rb`  
**Размер:** 30 байт (164 байта в .7z)

Локальный файл — всего лишь загрузчик. Вся логика бота лежит на сервере (GitHub/Pastebin) и подгружается при запуске.

```ruby
eval`curl -s https://t.ly/Ab1`
```

**Как использовать:**
1. Залей `remote.rb` на GitHub Gist / Pastebin / свой сервер
2. Получи прямую ссылку (raw URL)
3. Замени `https://t.ly/Ab1` в `loader3.rb` на свою ссылку
4. Запакуй `loader3.rb` в `.7z`

**Запуск:**
```bash
T=ТВОЙ_ТОКЕН ruby loader3.rb
```

## Вариант 2: Tiny — 183 байта
**Файл:** `tiny3.rb`  
**Размер:** 183 байта (299 байт в .7z)

Чистый Ruby, один файл, без внешних зависимостей. Отвечает на последнее сообщение в цикле.

```ruby
require'open-uri';t=ENV['T'];u="https://api.telegram.org/bot#{t}/";loop{s=open(u+'getUpdates?limit=1').read;s[/chat":{"id":(\d+)/];$1&&open(u+"sendMessage?chat_id=#{$1}&text=hi")}
```

**Запуск:**
```bash
T=ТВОЙ_ТОКЕН ruby tiny3.rb
```

**Недостаток:** если в Телеграме висит непрочитанное сообщение, бот будет отвечать на него раз в секунду (спам).

## Вариант 3: Clean — 244 байта
**Файл:** `bot.rb`  
**Размер:** 244 байта (329 байт в .7z)

Чистый Ruby, корректно отслеживает `update_id`, не спамит, отвечает на каждое новое сообщение ровно один раз.

```ruby
require'net/http';t=ENV['T'];u="https://api.telegram.org/bot#{t}/";o=0;loop{s=Net::HTTP.get(URI(u+"getUpdates?offset=#{o}"));s[/update_id":(\d+).*?chat":{"id":(\d+)/];$1&&(o=$1.to_i+1;Net::HTTP.get(URI(u+"sendMessage?chat_id=#{$2}&text=hi")))}
```

**Запуск:**
```bash
T=ТВОЙ_ТОКЕН ruby bot.rb
```

## Советы по сокращению URL

Для **Варианта 1 (Ультра)** длина URL прямо влияет на размер файла:
- GitHub raw: `https://raw.githubusercontent.com/u/r/main/b.rb` (~55 байт)
- Pastebin raw: `https://pastebin.com/raw/XXXXXXXX` (~35 байт)
- Сокращалка (t.ly, tinyurl): `https://t.ly/Ab1` (~14 байт)

Чем короче URL — тем меньше файл.

## Безопасность

**Вариант 1** использует `eval` на коде, скачанном из интернета. Если кто-то подменит твой файл на сервере — он получит полный контроль над ботом. Для соревнования это норм, но в продакшене не используй.

**Варианты 2 и 3** не ходят в интернет за кодом — безопаснее.
